Download Information

Gnofract 4D works on modern Unix-based operating systems such as Linux and FreeBSD. It can also be run on Mac OS X. It is not currently available for Windows.

There are a number of different ways to install Gnofract 4D depending on your operating system and/or distribution. If you're not sure which method to use, click the name of your system below to jump to the recommended method.

Compile from Source Code

Download this if you want to compile Gnofract 4D yourself, look at how it works, or you use an OS or distribution which doesn't use RPM files. (Or if you don't know what any of that means.) To compile, you will need header files for GTK2 and Python and a C++ compiler. Unpack the package with "tar xzvf gnofract4d-3.4.tar.gz" then see the README file for instructions on how to compile and install.

Source Package Latest stable source code package.
CVS Repository If you want to get (possibly unstable) up-to-the-minute source code, look here.
Binary RPM packages

Download one of these if you have a Linux distribution which uses RPM files running on an Intel or AMD computer. Examples are Fedora, SuSE, Mandrake/Mandriva.

Requires: Python 2.4, PyGTK 2.6+, g++
Optional: libjpeg-dev, libpng12-dev
Install the package using "rpm -i <rpmfile>".
RPM will tell you if any dependencies are missing when you run the installer.

Packages from elsewhere

Other people have kindly repackaged Gnofract 4D for additional Linux distributions and OSes. You may find these useful. However please note I don't maintain these and in most cases have never used them. They are often for an older release than the current one, especially if the new release has only just come out.
